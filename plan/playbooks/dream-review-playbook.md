# Dream Review Playbook

## Intent
Let agents and subagents imagine future Frikfrak directions without letting dream energy distort the current roadmap.

## Inputs
- plan/agent_self/my-agent1/dreams.md
- relevant reflective files in my-agent2
- current blueprints and progress state

## Review Loop
1. Read the active dream list.
2. Group ideas into:
- promote now
- park for later
- reject for current repo shape
3. Require one concrete repo anchor for any promoted dream.
4. Require one concrete constraint for any promoted dream.
5. Record the outcome in plan/history and the split ledgers.

## Promotion Rule
A dream may become active only if it has:
- a matching workload label
- one clear UI surface
- a bounded signal set
- a straightforward validation path

## Repo Examples
- Promote: hook-state teletext page using existing events.
- Park: fully autonomous gameified orchestration world.
- Reject for now: multi-network news aggregation with no orchestration tie-in.
