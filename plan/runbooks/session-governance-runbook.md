# Session Governance Runbook

## Purpose
Keep each Frikfrak work session deterministic from intake to closure.

## Steps
1. Read AGENTS.md.
2. Read plan/progress.md and plan/history.md.
3. Select one blueprint and one playbook.
4. Record the primary workload label.
5. Perform the task.
6. Update:
- plan/progress.md
- plan/history.md
- plan/heartbeat.md if the session meaningfully shifted direction
- prompt log file under plan/prompts-yyyymmdd_hhmmss.txt
- relevant split ledgers in plan/agent_self/

## Closure Criteria
A session closes cleanly when:
- changed docs agree on direction
- next checkpoint is explicit
- future work is narrowed, not broadened
