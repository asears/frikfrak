# Future Avenue: Card-Game Style VS Code Extension

## Why Later
The repo already has strong ingredients for a card-flavored UX: teletext visuals, agent event streams, WASM runtime, and multiplayer-style interaction ideas.

## Concept
A separate extension mode where coding actions map to cards:
- `Build cards`: compile, lint, test, wasm build
- `Insight cards`: diagnostics, AST map, hook stream
- `Action cards`: quick-fix templates, refactor routines
- `Lore cards`: project history, milestone unlocks

## Audience Signal
Kids and new users familiar with Pokemon/Magic/One Piece card metaphors may onboard faster with deck/battle/quest framing.

## Boundaries
- Keep this as a separate track from core Frikfrak runtime tasks.
- Maintain accessibility and non-gamified fallback workflows.

## First Exploration Tasks (Backlog)
- Draft card taxonomy aligned to real developer actions.
- Prototype one command palette deck view.
- Validate whether game framing improves retention/productivity.
